# *Recursion*

- Recursion is when a function calls itslef repeatedly.

- It is a method of solving computational problems where the solution depends on solution to smaller instances of same problem.

- **Going ahead with what actually recursion is we should have knowledge about the following topic:-**
    1. Iteration Loops - (Anything which can be done through loops can be achieved through Recursion. In Recursion we do function same function call repeatedly to make our work done.)
    - Now the questions can be solved with both loops and recursion but few questions have easy approach with loops and others can be easily solved by recursion depending on the situation.

    2. Function

- In recursion the same function goes on calling itself till a end-point where it no more further calls itself that end-point is Base case.

- The Logic of Implementing Recursion is Dividing the bigger problem into smaller problem and then solving that smaller problem and believing that the rest of the problem will be solved by the next call.
    Here we keep on dividing the bigger problem into smaller till a point where we already know that answer which is the base case.

### Recursion has Three important things as:-

1. Base Case
2. Self work
3. Assumption (That this work will automatically be done the function).

## *Stack Overflow*

- Just like leaving the terminating condition in the loop creates loop to run infinite times in a similar function calls have stack overflow.

- Stack Overflow occurs in one of the following ways:-
    1. Either we have used a large amount of local variables in the function stacks.
    2. Or missing a terminating (base case) in the Recursive function can lead to the stack overflow.

- We have a specific capacity of our stacks reaching those capacity can lead to the situation of staxck overflow.
